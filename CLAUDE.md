# Board Game Algorithm Visualization

ボードゲームの探索・学習アルゴリズムの動作をリアルタイムに可視化するWebアプリケーション。

## 作業の進め方

- **ファイルは1つずつ作成・編集し、その都度ユーザーの承認を得てから次に進む。まとめて複数ファイルを変更しない**
- 実装時はステアリングファイルの指示に従う

## 作業単位ドキュメント（ステアリングファイル）

作業は `.steering/[YYYYMMDD]-[開発タイトル]/` ディレクトリ単位で管理する。

### ルール

- **1ディレクトリ = 1作業単位**。1回のセッションで完了できる粒度に分割する
- ディレクトリ名の日付は作業の作成日。開発タイトルはケバブケースの短い名前（例: `backend-game-plugin`）
- 各ディレクトリには以下のファイルを置く:
  - `README.md` — 作業の概要、ゴール、完了条件、依存する作業、対象ファイル
- 実装時は「この作業のステアリングファイルに従って実装して」と指示する
- 作業完了後、README.md の冒頭に `status: done` を記載する
- 作業完了後、コミットの確認をする。コミットメッセージは日本語で記載する

### README.md のテンプレート

```markdown
# [作業タイトル]

status: todo

## ゴール
この作業で何を達成するか（1〜2文）

## 依存
- 前提となる作業ディレクトリ名（なければ「なし」）

## 完了条件
- [ ] 条件1
- [ ] 条件2

## 対象ファイル
- `path/to/file.ts` — 何をするか
- `path/to/another.py` — 何をするか

## 実装メモ
技術設計書の該当セクションへの参照、注意点など
```

## ドキュメント

- `docs/PRD.md` — プロダクト要件定義書
- `docs/technical-design.md` — 技術設計書（インターフェース定義、通信プロトコル、データフロー）
- `docs/conventions.md` — コーディング規約・技術リファレンス（技術スタック、アーキテクチャルール、命名規則）
